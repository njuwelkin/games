# 仙剑奇侠传 SDLPAL 脚本系统实现原理分析

## 概述

SDLPAL 脚本系统是仙剑奇侠传游戏的核心逻辑引擎，负责驱动游戏中的事件、对话、战斗和场景切换等所有交互行为。脚本以字节码形式存储在游戏数据文件中，由脚本执行器逐条解释执行。

---

## 一、脚本系统架构

### 1.1 整体架构

### 1.2 核心数据结构

#### 脚本条目结构

```c
typedef struct tagSCRIPTENTRY
{
   WORD          wOperation;     // 操作码 (16位)
   WORD          rgwOperand[3];  // 操作数 (最多3个，每个16位)
} SCRIPTENTRY, *LPSCRIPTENTRY;
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `wOperation` | WORD | 脚本操作码，决定执行何种操作 |
| `rgwOperand[0-2]` | WORD[3] | 操作数，具体含义取决于操作码 |

#### 事件对象结构

```c
typedef struct tagEVENTOBJECT
{
   SHORT        sVanishTime;         // 消失时间
   WORD         x, y;                // 地图坐标
   SHORT        sLayer;              // 层级
   WORD         wTriggerScript;      // 触发脚本入口
   WORD         wAutoScript;         // 自动脚本入口
   SHORT        sState;              // 对象状态 (隐藏/正常/阻挡)
   WORD         wTriggerMode;        // 触发模式
   WORD         wSpriteNum;          // 精灵编号
   // ... 其他字段
} EVENTOBJECT;
```

#### 场景结构

```c
typedef struct tagSCENE
{
   WORD         wMapNum;         // 地图编号
   WORD         wScriptOnEnter;  // 进入场景时执行的脚本
   WORD         wScriptOnTeleport;  // 传送出场景时执行的脚本
   WORD         wEventObjectIndex;  // 场景中事件对象的起始索引
} SCENE;
```

## 二、脚本执行机制

### 2.1 脚本类型

SDLPAL 脚本系统支持两种脚本类型：

| 类型 | 触发方式 | 执行频率 | 用途 |
|------|----------|----------|------|
| **Trigger Script** | 玩家触发（触摸/按键） | 单次执行 | 对话、事件、剧情触发 |
| **Auto Script** | 每帧自动执行 | 持续执行 | NPC移动、怪物AI |

### 2.2 脚本执行流程

```
开始执行
    │
    ▼
读取脚本条目 (SCRIPTENTRY)
    │
    ▼
解析操作码 (wOperation)
    │
    ├─── 流程控制类 (0x0000-0x000A)
    │       ├── 0x0000: 停止执行
    │       ├── 0x0003: 无条件跳转
    │       ├── 0x0004: 调用子脚本
    │       └── 0x0007: 开始战斗
    │
    ├─── 移动类 (0x000B-0x0016)
    │       ├── 0x000B-0x000E: NPC单步移动
    │       ├── 0x0010: NPC移动到指定位置
    │       └── 0x0070: 队伍移动到指定位置
    │
    ├─── 数据操作类 (0x0017-0x0033)
    │       ├── 0x001B: 增减HP
    │       ├── 0x001E: 增减金钱
    │       └── 0x001F: 添加物品
    │
    ├─── 战斗类 (0x003A-0x003B, 0x0060-0x006B)
    │       ├── 0x003A: 逃跑
    │       ├── 0x0060: 敌人即死
    │       └── 0x006A: 偷窃
    │
    ├─── 视觉/音效类 (0x0035, 0x0043, 0x0047, 0x0050-0x0054)
    │       ├── 0x0035: 屏幕震动
    │       ├── 0x0043: 设置背景音乐
    │       └── 0x0050: 淡出
    │
    ├─── 场景/地图类 (0x0059, 0x007F)
    │       ├── 0x0059: 切换场景
    │       └── 0x007F: 移动视口
    │
    └─── 对话类 (0x003B-0x003E, 0xFFFF)
            ├── 0x003B: 屏幕中央对话框
            ├── 0x003C: 屏幕上方对话框
            ├── 0x003D: 屏幕下方对话框
            └── 0xFFFF: 显示文本

    │
    ▼
执行操作 (调用对应功能函数)
    │
    ▼
更新脚本指针 (wScriptEntry++)
    │
    ▼
循环直到遇到停止指令 (0x0000)
```

### 2.3 核心执行函数

#### PAL_RunTriggerScript

```c
WORD PAL_RunTriggerScript(WORD wScriptEntry, WORD wEventObjectID)
```

**功能**：执行触发脚本

**参数**：
- `wScriptEntry`: 脚本入口地址
- `wEventObjectID`: 触发脚本的事件对象ID（0xFFFF表示继承上次的对象）

**执行流程**：

1. 初始化脚本成功标志 `g_fScriptSuccess = TRUE`
2. 设置对话默认速度
3. 主循环：读取脚本条目 → 解析操作码 → 执行操作 → 更新脚本指针
4. 遇到 `0x0000`（停止）或 `0x0001`（停止并替换入口）时退出

#### PAL_InterpretInstruction

```c
static WORD PAL_InterpretInstruction(WORD wScriptEntry, WORD wEventObjectID)
```

**功能**：解释并执行单条脚本指令

**返回值**：下一条要执行的脚本入口地址

**设计特点**：
- 通过 `switch-case` 分发操作码
- 支持条件跳转（通过修改 `wScriptEntry` 实现）
- 支持状态反馈（通过 `g_fScriptSuccess` 全局变量）

## 三、操作码分类详解

### 3.1 流程控制类（0x0000-0x000A）

| 操作码 | 名称 | 功能说明 | 操作数 |
|--------|------|----------|--------|
| 0x0000 | STOP | 停止脚本执行 | 无 |
| 0x0001 | STOP_NEXT | 停止并指向下一条 | 无 |
| 0x0002 | STOP_COND | 条件停止（帧数检查） | [跳转地址, 帧数] |
| 0x0003 | JUMP | 无条件跳转 | [跳转地址, 帧数] |
| 0x0004 | CALL | 调用子脚本 | [脚本地址, 对象ID] |
| 0x0005 | REDRAW | 重绘屏幕 | [是否清屏, 延迟帧数, 更新队伍] |
| 0x0006 | JUMP_RANDOM | 概率跳转 | [概率%, 跳转地址] |
| 0x0007 | BATTLE | 开始战斗 | [敌人组ID, 失败跳转, 逃跑跳转] |
| 0x0008 | NEXT | 指向下一条 | 无 |
| 0x0009 | WAIT | 等待指定帧数 | [帧数, 是否更新, 更新队伍] |
| 0x000A | CONFIRM | 确认菜单 | [取消时跳转地址] |

## 四、脚本触发机制

### 4.1 触发模式

事件对象支持多种触发模式：

```c
typedef enum tagTRIGGERMODE
{
   kTriggerNone                  = 0,  // 无触发
   kTriggerSearchNear            = 1,  // 近距离搜索触发
   kTriggerSearchNormal          = 2,  // 中距离搜索触发
   kTriggerSearchFar             = 3,  // 远距离搜索触发
   kTriggerTouchNear             = 4,  // 近距离接触触发
   kTriggerTouchNormal           = 5,  // 中距离接触触发
   kTriggerTouchFar              = 6,  // 远距离接触触发
   kTriggerTouchFarther          = 7,  // 更远距离接触触发
   kTriggerTouchFarthest         = 8   // 最远距离接触触发
} TRIGGERMODE;
```

### 4.2 触发流程

```
玩家移动/按键
    │
    ▼
检测事件对象
    │
    ├── 搜索触发: 检测玩家与对象的距离
    │       └── 如果在触发范围内 → 执行触发脚本
    │
    └── 接触触发: 检测玩家与对象的接触
            └── 如果接触 → 执行触发脚本
```

### 4.3 自动脚本执行

自动脚本每帧执行一次，用于实现：
- NPC巡逻移动
- 怪物追逐玩家
- 场景动态效果

```c
WORD PAL_RunAutoScript(WORD wScriptEntry, WORD wEventObjectID)
```

自动脚本与触发脚本共享相同的操作码集，但执行方式不同：
- **触发脚本**：完整执行直到遇到停止指令
- **自动脚本**：每帧执行一条指令（非阻塞）

## 五、脚本数据存储

### 5.1 脚本文件格式

脚本数据存储在 `SSS.MKF` 文件中，采用 MKF 格式封装。每个脚本条目占用固定大小（8字节：1字节操作码 + 3×2字节操作数）。

### 5.2 脚本入口地址

脚本入口地址以偏移量形式存储在：
- `EVENTOBJECT.wTriggerScript` - 触发脚本入口
- `EVENTOBJECT.wAutoScript` - 自动脚本入口  
- `SCENE.wScriptOnEnter` - 场景进入脚本
- `SCENE.wScriptOnTeleport` - 场景传送脚本
- `OBJECT.item.wScriptOnUse` - 物品使用脚本

## 六、设计特点与技术亮点

### 6.1 设计特点

| 特点 | 说明 |
|------|------|
| **栈式指令架构** | 操作码+操作数的经典设计，易于扩展 |
| **双脚本系统** | 触发脚本+自动脚本分离，各司其职 |
| **状态反馈机制** | 通过 `g_fScriptSuccess` 传递执行结果 |
| **条件跳转支持** | 丰富的条件判断操作码，支持复杂逻辑 |
| **事件驱动** | 基于事件对象的触发机制 |

### 6.2 技术亮点

1. **非阻塞移动**：移动指令通过返回当前脚本地址实现逐帧移动，不阻塞游戏主线程
2. **脚本调用机制**：支持脚本嵌套调用，通过 `CALL` 指令实现代码复用
3. **概率系统**：支持随机跳转和概率事件，增加游戏随机性
4. **条件分支**：丰富的条件判断操作码，支持复杂剧情逻辑

## 七、脚本系统工作示例

### 示例：NPC对话流程

```
脚本序列:
0x003D 0001 0000 0001    ; 显示对话框(下方)
0xFFFF 0042               ; 显示文本ID 0x42 ("你好，旅行者！")
0x0005 0001 0001 0000    ; 重绘屏幕，等待1帧
0x000A 0008               ; 确认菜单，取消跳转到0x0008
0xFFFF 0043               ; 显示文本ID 0x43 ("需要帮助吗？")
0x0000                    ; 停止

执行流程:
1. 显示对话框
2. 显示对话文本
3. 等待按键
4. 显示确认菜单
5. 如果玩家取消，跳转到0x0008
6. 继续显示后续文本
7. 脚本结束
```

### 示例：战斗触发

```
脚本序列:
0x0007 0003 000A 0000    ; 开始战斗(敌人组3)，失败跳转到0x000A
0xFFFF 0050               ; 显示胜利文本
0x0001                    ; 停止

0x000A:
0xFFFF 0051               ; 显示失败文本
0x004E                    ; 读取存档

执行流程:
1. 触发战斗
2. 如果战斗失败，跳转到失败处理
3. 胜利则显示胜利文本
4. 失败则显示失败文本并读取存档
```

## 八、总结

SDLPAL 脚本系统是一个设计精巧的字节码解释器，具有以下特点：

1. **完整的指令集**：覆盖游戏所有核心功能，包括移动、战斗、对话、物品等
2. **灵活的执行机制**：支持触发脚本和自动脚本两种模式
3. **事件驱动架构**：基于事件对象的触发机制，易于扩展
4. **条件分支支持**：丰富的条件判断操作码，支持复杂剧情逻辑
5. **非阻塞设计**：移动等耗时操作采用逐帧执行，保证游戏流畅性

该脚本系统是仙剑奇侠传游戏逻辑的核心，通过脚本驱动整个游戏的剧情推进和玩家交互。
